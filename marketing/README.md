# marketing/ — self-hosted marketing & analytics stack

All apps here share the external **`marketing`** Docker network, sit behind the
cloudflared tunnel (`drawbridge`), and expose their DB/cache metrics to Prometheus
(`observatory`). Each app keeps its own private `<app>-internal` network for its
datastore.

> **Status: defined but STOPPED.** These were authored for review — nothing here is
> running. Bring an app up only after sizing the box (PostHog especially is heavy).
> Each needs its `.env` filled first (copy from `.env.example`).

| App | What | Local port | DB / deps | Metrics |
|-----|------|-----------|-----------|---------|
| [postiz](./postiz) | Social scheduling (Buffer/Hootsuite) | 5050 | Postgres, Redis, Temporal+ES | pg, redis, temporal pg+es exporters |
| [listmonk](./listmonk) | Newsletters / mailing lists | 8410 | Postgres | postgres_exporter |
| [umami](./umami) | Privacy-friendly web analytics | 8411 | Postgres | postgres_exporter |
| [matomo](./matomo) | Web analytics | 8412 | MariaDB | mysqld_exporter |
| [metabase](./metabase) | BI / dashboards | 8413 | Postgres | postgres_exporter |
| [typebot](./typebot) | Chatbot / form builder + viewer | 8414/8415 | Postgres | postgres_exporter |
| [chatwoot](./chatwoot) | Customer engagement / shared inbox | 8416 | Postgres (pgvector), Redis | pg + redis exporters |
| [plausible](./plausible) | Privacy-first analytics | 8417 | Postgres, ClickHouse | pg_exporter + CH native `:9363` |
| [posthog](./posthog) | Product analytics (heavy) | 8418 | Postgres, Redis, ClickHouse, Kafka, MinIO | pg + redis exporters + CH native |

Local ports are bound to `127.0.0.1` for testing only — public access is via the
cloudflared tunnel (`<app>.itproxima.com`).

## Observability wiring
- **Prometheus** (`observatory/prometheus/prometheus.yml`): every exporter above has a
  scrape target. They show **DOWN** until the corresponding stack is started — that's
  expected.
- **Grafana** (`observatory/grafana/`): dashboards are generated by `gen_dashboards.py`
  — PostgreSQL/Redis per-instance dashboards plus dedicated **MySQL** and **ClickHouse**
  dashboards. Regenerate with `python3 observatory/grafana/gen_dashboards.py`.

## Apply one app
```bash
cd marketing/<app>
cp .env.example .env && $EDITOR .env      # fill secrets
docker compose up -d
```
